# 产品需求文档 (PRD) — Pro-OW 守望先锋社区论坛

> 版本: v1.0 | 日期: 2026-05-29 | 状态: Draft

---

## 一、产品概述

### 1.1 产品定位

Pro-OW 是面向守望先锋玩家的新一代社区平台。区别于传统论坛，Pro-OW 深度集成游戏数据，提供 AI 复盘、实时组队、电竞竞猜等创新功能，打造「游戏内体验 → 社区沉淀 → 成长闭环」的完整生态。

### 1.2 目标用户

| 用户画像 | 特征 | 核心需求 |
|---|---|---|
| 竞技玩家 | 关注段位、胜率、英雄数据 | 复盘分析、攻略学习、高分段交流 |
| 休闲玩家 | 喜欢创意工坊、自定义模式 | 模式分享、轻松社交、趣图分享 |
| 电竞观众 | 关注 OWL 赛事 | 赛事资讯、竞猜互动、选手讨论 |
| 内容创作者 | 产出攻略/视频/直播 | 曝光渠道、粉丝互动、个人品牌 |
| 车队队长 | 经常组队开黑 | 找队友、约战、战队管理 |

### 1.3 核心差异化

- **游戏数据绑定**：战网账号绑定，自动同步段位 / 英雄数据
- **AI 复盘**：上传录像代码，AI 分析走位与决策
- **实时组队大厅**：按位置 / 段位即时匹配开黑队友
- **社区赛季制**：论坛活跃度排名，赛季末发放奖励

## 二、功能模块

### 2.1 用户系统 (P0)

**2.1.1 注册与登录**
- 邮箱注册 + 密码登录
- 战网 OAuth 绑定（后续版本）
- JWT Token + Refresh Token 鉴权

**2.1.2 个人主页**
- 头像、昵称、签名、个人简介
- 战网数据卡片（段位、常用英雄、胜率）
- 发帖 / 评论历史
- 获得的徽章展示

**2.1.3 用户等级与成就**
- 经验值系统（发帖、评论、被赞、每日签到）
- 等级称号（青铜→宗师，对应游戏段位）
- 徽章系统（英雄专精、赛季活跃、助人为乐等）

**2.1.4 权限体系**
- 角色：普通用户 / 版主 / 超级管理员
- 权限：禁言、删帖、置顶、板块管理

**用户故事：**
> 作为一个竞技玩家，我想绑定我的战网账号，以便在论坛展示我的游戏段位和英雄数据。

> 作为一个新用户，我想通过邮箱注册并完善个人资料，以便融入社区。

---

### 2.2 内容系统 (P0)

**2.2.1 板块分类**
- 综合讨论
- 英雄攻略（按英雄分类的子板块）
- 赛事资讯
- 组队开黑
- 创意工坊
- 灌水区

**2.2.2 发帖功能**
- 富文本编辑器（Markdown + 所见即所得）
- 支持插入图片（多图上传 + 拖拽排序）
- 支持插入视频（外链 YouTube/Bilibili 自动嵌入）
- 帖子标签（最多 5 个）
- 帖子类型：普通帖 / 投票帖 / 图集帖

**2.2.3 帖子管理**
- 编辑 / 删除（软删除）
- 草稿自动保存
- 发布前预览

**2.2.4 内容可见性**
- 公开帖：所有人可见
- 仅粉丝可见（后续）

**用户故事：**
> 作为一个攻略作者，我想发布带图片和视频的英雄攻略，以便帮助其他玩家提升水平。

> 作为一个版主，我想审核帖子内容并有权删除违规帖，以维护社区秩序。

---

### 2.3 互动系统 (P0)

**2.3.1 评论系统**
- 一级评论 + 楼中楼回复（二级）
- 评论富文本（支持图片）
- 评论排序：按时间 / 按热度

**2.3.2 社交互动**
- 点赞（帖子 + 评论）
- 收藏帖子（私有收藏夹）
- 分享（生成分享卡片，复制链接）

**2.3.3 消息通知**
- 通知类型：被回复、被点赞、被关注、系统通知
- 通知渠道：站内信 + 邮件（可选）
- 通知设置：可关闭特定类型通知

**用户故事：**
> 作为一个用户，我想在帖子下评论并收到回复通知，以便参与讨论。

> 作为一个用户，我想收藏有价值的攻略帖，方便以后回顾。

---

### 2.4 搜索与发现 (P0)

**2.4.1 全文搜索**
- 帖子标题 + 内容全文检索
- 按板块 / 标签 / 时间范围筛选
- 搜索结果高亮 + 排序

**2.4.2 首页信息流**
- 排序：最新 / 最热 / 精华
- 个性化推荐（基于浏览历史和互动数据，后续）

**用户故事：**
> 作为一个用户，我想搜索有关「源氏连招」的攻略帖，以便快速找到需要的内容。

---

### 2.5 实时组队 (P1)

**2.5.1 组队大厅**
- 发布组队信息：位置需求、段位要求、模式（竞技/快速）
- 筛选条件：段位范围、位置（Tank/DPS/Support）
- 一键加入队伍

**2.5.2 实时聊天**
- 临时队伍聊天室
- WebSocket 实时通信

**用户故事：**
> 作为一个坦克玩家，我想在组队大厅找到缺坦克的车队，以便快速开黑。

---

### 2.6 赛事竞猜 (P1)

**2.6.1 竞猜创建**
- 管理员发布竞猜事件（OWL 比赛、世界杯）
- 竞猜选项：胜负、比分、MVP 等

**2.6.2 竞猜参与**
- 用户投注论坛积分
- 实时赔率展示
- 赛后自动结算

**用户故事：**
> 作为一个电竞粉丝，我想为 OWL 比赛结果下注，赢取论坛积分以兑换奖励。

---

### 2.7 创意工坊 (P1)

**2.7.1 代码分享**
- 上传 / 导入自定义模式代码
- 自动解析代码中的英雄、模式类型
- 用户评分 + 评论

**2.7.2 热门排行**
- 本周最热 / 本月最热
- 官方推荐

**用户故事：**
> 作为一个创意工坊作者，我想分享我的自定义模式代码并查看下载量，以获得成就感。

---

### 2.8 AI 复盘 (P2)

**2.8.1 录像上传**
- 上传录像代码
- 自动解析录像元数据（英雄、地图、时长）

**2.8.2 AI 分析**
- 走位热力图
- 技能使用分析
- 关键决策点标注
- AI 生成改进建议

**用户故事：**
> 作为一个竞技玩家，我想上传比赛录像让 AI 分析我的问题，以便针对性提升。

---

### 2.9 论坛赛季 (P2)

**2.9.1 赛季机制**
- 论坛赛季与游戏赛季同步
- 赛季积分：发帖、评论、被赞、精选等行为加权计分

**2.9.2 赛季奖励**
- 专属头像框
- 赛季称号
- 虚拟货币

**用户故事：**
> 作为一个活跃用户，我想在赛季末获得专属奖励，以证明我的社区贡献。

## 三、业务规则

### 3.1 积分规则

| 行为 | 积分 |
|---|---|
| 发帖 | +10 |
| 评论 | +3 |
| 帖子被赞 | +2 |
| 评论被赞 | +1 |
| 帖子被精选 | +50 |
| 每日签到 | +5 |
| 帖子被删 | -20 |

### 3.2 等级规则

| 等级 | 称号 | 所需经验 |
|---|---|---|
| 1 | 青铜 | 0 |
| 5 | 白银 | 200 |
| 10 | 黄金 | 500 |
| 15 | 白金 | 1000 |
| 20 | 钻石 | 2000 |
| 25 | 大师 | 5000 |
| 30 | 宗师 | 10000 |
| 35 | 五百强 | 20000 |

### 3.3 内容审核规则

- 新用户前 5 条帖子需人工审核
- 敏感词过滤（自动替换为 ***）
- 被举报超过 3 次自动隐藏，待版主审核
- 同一用户 1 分钟内最多发 2 条帖子（防刷屏）

### 3.4 竞猜规则

- 比赛开始前 10 分钟停止下注
- 每人每场最多下注 1000 积分
- 赔率动态调整（基于下注分布）
- 结算延迟：比赛结束后 30 分钟内人工确认

## 四、非功能需求

### 4.1 性能

- 帖子详情页首屏加载 < 2s
- 搜索响应时间 < 500ms
- WebSocket 消息延迟 < 100ms
- 支持 10 万 DAU（设计目标）

### 4.2 安全

- HTTPS 全站加密
- 接口防刷（IP + User 级别限流）
- XSS / CSRF / SQL 注入防护
- 文件上传类型 + 大小校验

### 4.3 可用性

- 核心服务可用性 >= 99.9%
- 数据每日全量备份
- 灰度发布 + 快速回滚能力

### 4.4 可扩展性

- 微服务独立部署、独立扩缩
- 消息队列异步解耦
- 读写分离（Elasticsearch 读，PostgreSQL 写）
